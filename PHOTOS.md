# Фотоархив AidaCamp — инструкция для агентов

Весь фотоархив AidaCamp проиндексирован и размечен AI (Gemini Vision). Каталог содержит **~9200 фотографий** с описаниями, сценами, тегами и оценкой качества.

## Где лежат данные

| Файл | Расположение | Что содержит |
|------|-------------|--------------|
| `photo_catalog.json` | `scripts/photo_catalog.json` | Полный каталог: 9199 фото с AI-разметкой |
| `photo_catalog_summary.json` | `scripts/photo_catalog_summary.json` | Сводка: статистика + топ-фото по сценам |
| `disk_index.json` | `scripts/disk_index.json` | Индекс всех файлов на диске (фото + документы + видео) |
| `disk_index.csv` | `~/Downloads/disk_index.csv` | То же в CSV для Excel/Google Sheets |

## Структура записи в photo_catalog.json

```json
{
  "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic": {
    "сцена": "хакатон",
    "возраст": "11-14",
    "качество": "соцсети",
    "люди": true,
    "описание": "Молодой человек выступает с презентацией перед аудиторией детей.",
    "теги": ["презентация", "выступление", "дети", "хакатон", "образование"],
    "_путь": "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic",
    "_папка": "Медиа/2024/Фото/Прочее",
    "_год": "2024",
    "_имя": "IMG_2449.heic"
  }
}
```

### Поля

| Поле | Значения | Описание |
|------|----------|----------|
| `сцена` | занятие, хакатон, эмоции, спорт, территория, столовая, вожатые, награждение, портрет, обучение, скринкаст, обзор, мероприятие, прочее, документ | Тип сцены на фото |
| `качество` | **сайт** (лучшее), **соцсети** (хорошее), **архив** (низкое) | Оценка качества для использования в контенте |
| `возраст` | 7-10, 11-14, смешанный, нет_детей | Возрастная группа детей на фото |
| `люди` | true / false | Есть ли люди на фото |
| `описание` | строка | Краткое описание сцены на русском |
| `теги` | массив строк | 3-5 ключевых слов |

### Качество — что означает

- **сайт** (2222 фото) — высокое качество, подходит для главной страницы, лендингов, печати
- **соцсети** (4964 фото) — хорошее качество, подходит для постов в соцсетях, сторис, рассылок
- **архив** (1987 фото) — низкое качество или нерелевантные, для внутреннего использования

## Скрипт-утилита yadisk.sh

Для работы с каталогом и API Яндекс.Диска есть готовый скрипт:

```bash
# Путь: scripts/yadisk.sh

# Поиск фото по описанию/тегам
./scripts/yadisk.sh search "дети программирование"

# Лучшие фото по сцене (сортировка: сайт → соцсети → архив)
./scripts/yadisk.sh best занятие 5
./scripts/yadisk.sh best хакатон 10

# Список всех сцен с количеством
./scripts/yadisk.sh scenes

# Общая статистика
./scripts/yadisk.sh stats

# Прямая ссылка на скачивание файла (нужен YADISK_TOKEN)
./scripts/yadisk.sh url "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic"

# Ссылка на превью (маленькая версия)
./scripts/yadisk.sh preview "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic"

# Метаданные файла
./scripts/yadisk.sh info "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic"
```

### Переменные окружения

- `YADISK_TOKEN` — OAuth-токен Яндекс.Диска. Нужен только для команд `url`, `preview`, `info`. Для `search`, `best`, `scenes`, `stats` токен не нужен — они работают с локальным JSON.

## MCP-инструмент photos

В MCP-сервере (`aidacamp-tools`) доступен инструмент `photos`:

```
photos(command: "search", query: "дети программирование")
photos(command: "best", query: "занятие", count: 5)
photos(command: "scenes")
photos(command: "stats")
photos(command: "url", query: "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic")
photos(command: "preview", query: "disk:/Медиа/2024/Фото/Прочее/IMG_2449.heic")
```

## Как получить ссылку на фото

Яндекс.Диск не даёт постоянных прямых ссылок. Для скачивания:

1. Вызови `yadisk.sh url "disk:/путь/к/файлу"` — получишь временную ссылку (~30 мин)
2. Скачай файл по этой ссылке (curl/wget)

Для превью (маленькая версия для предпросмотра):
1. Вызови `yadisk.sh preview "disk:/путь/к/файлу"` — получишь ссылку на уменьшенную версию

### Yandex.Disk REST API (справка)

Если нужно что-то за пределами скрипта:

```bash
# Скачать файл
curl -s -H "Authorization: OAuth $YADISK_TOKEN" \
  "https://cloud-api.yandex.net/v1/disk/resources/download?path=ENCODED_PATH" \
  | jq -r '.href'

# Получить превью
curl -s -H "Authorization: OAuth $YADISK_TOKEN" \
  "https://cloud-api.yandex.net/v1/disk/resources?path=ENCODED_PATH&fields=preview&preview_size=L" \
  | jq -r '.preview'

# Метаданные файла
curl -s -H "Authorization: OAuth $YADISK_TOKEN" \
  "https://cloud-api.yandex.net/v1/disk/resources?path=ENCODED_PATH"
```

Путь нужно URL-encode: `disk:/Медиа/2024/...` → `disk%3A%2F%D0%9C%D0%B5%D0%B4%D0%B8%D0%B0%2F2024%2F...`

## Типичные задачи

### Подобрать фото для поста в соцсети

```bash
# 1. Найти по теме
./scripts/yadisk.sh search "дети счастливые программирование"

# 2. Или взять лучшие по сцене
./scripts/yadisk.sh best занятие 5

# 3. Получить ссылку на скачивание
./scripts/yadisk.sh url "disk:/Медиа/2024/Фото/Прочее/IMG_1234.jpg"
```

### Подобрать фото для сайта

```bash
# Фото качества "сайт" — лучшие
./scripts/yadisk.sh best обучение 10
# Первые результаты будут с качеством "сайт"
```

### Найти фото определённого возраста

```bash
# Поиск в JSON напрямую
python3 -c "
import json
c = json.load(open('scripts/photo_catalog.json'))
for p, e in c.items():
    if not isinstance(e, dict): continue
    if e.get('возраст') == '7-10' and e.get('качество') == 'сайт':
        print(f\"{e.get('сцена')}: {p}\")
        print(f\"  {e.get('описание')}\")
" | head -20
```
